Design Notes — EventLobby

Layout

Centered column, max-width: 520px, padded top 40px to clear the devNav bar. align-items: center on the flex column keeps all sections centred. The action row uses margin-top: auto to push itself to the natural bottom.

Event header — gold accent line

The event name block uses border-left: 3px solid var(--accent) with a left-padding indent. This matches the ShootingHUD's gold-on-dark aesthetic without adding a visual element that competes with the event name typography. The flag + country name sit directly below, muted, so the hierarchy is: name → origin.

Distance cards — colour formula

Distances are shown in shot order (shortest → longest, per GDD): 18m, 70m, 90m. Colour uses the same helper as ShootingHUD: hsl(120*(1 − (m−18)/72), 70%, 62%).

  • 18m → hsl(120, 70%, 62%) — green
  • 70m → hsl(33, 70%, 62%) — orange
  • 90m → hsl(0, 70%, 62%) — red

The border of each card uses the same colour, keeping it consistent with ShootingHUD's distanceCard.

Season standings — 5-column grid

Grid: 1.6rem 1.6rem 1fr 2rem 2.8rem → rank | flag | name | pts-label | pts-value. The pts-label column ("pts") uses a very muted colour to act as a subtle data annotation without competing with the value. The player row gets border-left: 2px solid var(--accent) and a 5% gold tint background — same approach as ShootingHUD's lbPlayer.

Separator state: when the player is outside top 5, a ··· separator row is inserted between the top 5 and the pinned player row. The separator row uses the same grid padding and a very muted colour to feel like a continuation, not a divider.

Action row

"Career" is a ghost/outlined button — muted border, muted text, low visual weight. "Start Event →" is the primary CTA using var(--accent) fill with dark text, bold, larger padding. Justified to opposite ends of the max-width container.

Season context

"Season 1 · Event 3 of 5" sits absolutely positioned top-right, using var(--font-mono) and var(--muted). Small (0.7rem) so it reads as metadata, not a heading.

Implementation gaps

  • Data is hard-coded; real implementation needs GameState props: eventName, hostCountry, distances, standings, playerRank, seasonNum, eventIndex, totalEvents
  • Career button href is a placeholder; needs router integration (useNavigate or Link)
  • Start Event button needs onClick handler to transition to ShootingHUD
  • The separator / player-pin logic: filter top 5; if player rank > 5, show separator + player row
